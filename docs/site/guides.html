Platform features

Reference behavior for attachments, embeds, invites, badges, and boost perks.

Attachments and embeds

  • Attachments use multipart upload via /v1/attachments/upload.
  • Tier limits are applied server-side and include boost perks (up to 100 MB for qualified users).
  • Messages can include structured embeds and link previews.
  • Extension commands can return embed payloads the same way as normal message API flows.

Invites and join flow

  • Join accepts invite code or full URL in one input.
  • Opening invite links should resolve invite metadata and prompt user confirmation before joining.
  • Invite events can emit richer message/embed history entries in the server feed.
  • Boost-enabled servers can request custom permanent invite aliases when permitted by policy.

Badges and boost perks

  • Platform owner badge: crown icon style.
  • Platform admin badge: hammer icon style.
  • Boost badge: plus icon with tiered background colors (2+ year tier supports gold background).
  • UI should expose badge name on hover tooltip.

Server branding

  • Server logo and banner fields are editable via profile routes.
  • Image validation should reject invalid formats/sizes and return actionable errors.
