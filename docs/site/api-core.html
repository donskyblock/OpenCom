API

Core API reference

Core handles identity, sessions, server registry, invites, social, billing, profile metadata, and extension orchestration.

Authentication

MethodPathPurpose
POST/v1/auth/registerCreate account
POST/v1/auth/loginIssue access + refresh tokens
POST/v1/auth/refreshRotate access token
GET/v1/auth/sessionsList active sessions
DELETE/v1/auth/sessions/:sessionIdRevoke one session

Servers and membership

MethodPathPurpose
POST/v1/serversCreate server registration
GET/v1/serversList user servers
PATCH/v1/servers/:serverId/profileUpdate server logo/banner/profile
POST/v1/servers/reorderReorder server list
POST/v1/servers/:serverId/membership-tokenRefresh membership token

Invites

MethodPathPurpose
POST/v1/invitesCreate invite (supports custom/permanent when allowed)
GET/v1/invites/:codeResolve invite preview/metadata
POST/v1/invites/joinJoin via code or URL input
POST/v1/invites/:code/joinJoin via explicit invite code

Extensions from Core

MethodPathPurpose
GET/v1/extensions/catalogGlobal extension registry
GET/v1/servers/:serverId/extensionsServer extension state
GET/v1/servers/:serverId/extensions/:extensionId/configResolved extension config
PUT/v1/servers/:serverId/extensions/:extensionId/configUpdate extension config in realtime
POST/v1/servers/:serverId/extensions/:extensionIdEnable/disable extension

Other route groups

  • /v1/social/* friends, requests, DM threads/messages
  • /v1/profile-images/* profile image hosting
  • /v1/billing/boost/* boost status and checkout sync
  • /v1/admin/* platform admin and badges

Source routes live in backend/packages/core/src/routes.
