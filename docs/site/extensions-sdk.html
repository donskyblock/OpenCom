Extensions

OpenCom Extension SDK

Build server extensions with command registration, persisted config, runtime lifecycle hooks, and high-level API clients.

Manifest

{
  "id": "my-extension",
  "name": "My Extension",
  "author": "you",
  "version": "1.0.0",
  "scope": "server",
  "entry": "index.js",
  "permissions": ["all"],
  "configDefaults": { "featureEnabled": true }
}

Config system

import { command, defineConfig } from "opencom-extension-sdk";

export const configDefaults = defineConfig({
  pingPrefix: "pong"
});

export const commands = [
  command({
    name: "ping",
    async execute(ctx) {
      const cfg = await ctx.config.get();
      return { content: `${cfg.pingPrefix} ${ctx.userId}` };
    }
  })
];

Realtime lifecycle

Extensions can be activated/deactivated without node restarts. Command availability updates immediately when extension state changes.

SDK API helpers

  • apis.servers.refreshMembershipToken(serverId)
  • apis.servers.reorder(serverIds)
  • apis.invites.joinFromInput(codeOrUrl)
  • apis.nodeGuilds.kickMember(...) / banMember(...) / unbanMember(...)
  • apis.nodeGuilds.reorderChannels(guildId, items)
  • apis.attachments.upload(file, options)
  • apis.extensions.serverConfig(serverId, extensionId)
  • apis.extensions.setServerConfig(serverId, extensionId, config)

Config endpoints

  • GET /v1/servers/:serverId/extensions/:extensionId/config
  • PUT /v1/servers/:serverId/extensions/:extensionId/config
